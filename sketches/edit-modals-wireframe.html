Design Pattern Summary
Module Edit Trigger Edit Style Deactivate Locked Fields
1. Invoice Edit/Delete buttons in detail modal footer Three-mode modal (view → edit → delete) Cascades to payments Direction, partner, currency, project
2. Entity Edit/Deactivate in detail panel header Three-mode modal Hidden from dropdowns Type, doc type, doc number
3. Contact Pencil icon on contact row Inline row edit (blue background) Existing trash icon None
4. Project Edit button in project header Three-mode modal N/A (use status → cancelled) Code, type, currency
5. Partner Share Click on % cell Inline cell edit (like budget) Existing trash icon Partner name (edit share only)
Shared patterns across all modals:
  • Action icons — Always pencil (✎) for edit + red trash can for delete, side by side. Used consistently everywhere: table rows, modal footers, inline actions
  • Three-mode modal — View (detail + pencil/trash buttons) → Edit (locked + editable fields) → Delete (dimmed content + red confirmation box)
  • Locked fields — Fields that would break referential integrity. Shown with 🔒 lock icon and zinc-400 text
  • Inline edit — Used for simple single-field edits (contacts, profit share, budget). No modal needed — edit in place with Save button
  • Soft delete language — Always "Deactivate" not "Delete". Confirmation shows impact (related records affected). "Can be reversed by admin."
  • Server actions — Each modal calls a dedicated server action (updateInvoice, updateEntity, updateProject, etc.) matching the existing updatePayment/deactivatePayment pattern
