Design Notes — Payments Redesign

Problem 1: Summary buried at the bottom + too cramped

The current sticky footer jams In/Out/Net into a single line of text. Hard to parse.

Fix: Compact summary row inside the table footer — same location but properly formatted with color-coded labels, monospaced amounts, and clear separation. No cards at top — this is a browse page, not a dashboard. Financial Position already covers the dashboard view.

Problem 2: No date range control

Summary says "Last 30 Days" but the user can't change it.

Fix: Date range picker inline with all other filters in a single row. No preset buttons — just a clean date range selector. Summary and table both respond to the selected range.

Problem 3: Direction separated from Amount

The In/Out badge sits at the far right of the row, 8 columns away from the amount. Users must zig-zag their eyes across the entire row.

Fix: Merge direction into the amount column using signed values: +S/ 3,720.00 (green) for inbound, −$3,500.00 (red) for outbound. The Direction column is eliminated entirely.

Problem 4: 8 equal-weight columns

Date, Related, Entity, Project, Type, Bank, Amount, Dir — all visually equal. The eye doesn't know where to anchor.

Fix: Collapse to 5 columns by merging Related + Project + Type badge into a single "Reference" column. This groups related metadata (what document, which project, what type) together. Type badge only appears for Detraccion/Retencion — Regular is the default and needs no label.

Problem 5: Filters hidden behind a button

The most common operations (filter by direction, by type) require clicking "Filters" to reveal a panel.

Fix: Always-visible filter bar with toggle chips for direction (In/Out), payment type, and related-to, plus compact dropdowns for project/bank and inline search. All filters are one-click — nothing hidden.
